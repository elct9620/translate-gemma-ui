# TranslateGemma UI

## Purpose

提供一個本地運行的翻譯工具，使用開放權重模型 TranslateGemma 4B，讓使用者在不依賴雲端服務的情況下完成多語言翻譯與字幕在地化工作。

## Users

| 使用者 | 需求 |
|--------|------|
| 影音工作者 | 翻譯 SRT 字幕檔，維持詞彙一致性 |
| 字幕翻譯者 | 批次翻譯字幕，搭配詞彙表控制譯詞 |
| 一般使用者 | 簡單的文字段落翻譯 |

使用者不一定具備工程背景，安裝與操作流程需簡單直覺。

## Impacts

- 翻譯過程完全在本地執行，保護內容隱私
- 透過詞彙表確保翻譯用詞一致
- 降低對付費翻譯 API 的依賴

## Success Criteria

- 使用者可在 macOS 或 Windows 上完成安裝並啟動應用程式
- 使用者可透過 GUI 完成一份 SRT 字幕檔的翻譯並下載結果
- 在 4GB VRAM 的 GPU 上，系統可載入模型並完成翻譯
- 詞彙表中指定的譯詞出現在翻譯結果中

## IS / IS NOT

| IS | IS NOT |
|----|--------|
| 本地桌面應用程式 | 雲端翻譯服務 |
| 批次字幕翻譯工具 | 即時語音翻譯 |
| 使用單一模型（TranslateGemma 4B） | 多模型切換平台 |
| 支援 SRT 格式 | 通用字幕格式轉換器 |

## Features

1. **文字翻譯**：使用者輸入文字，選擇來源與目標語言，系統輸出翻譯結果
2. **SRT 字幕翻譯**：使用者上傳 SRT 檔案，系統逐句翻譯並保留時間軸，輸出翻譯後的 SRT 檔案
3. **詞彙表**：使用者提供詞彙對照表，系統在翻譯時優先採用指定譯詞
4. **GUI 介面**：透過瀏覽器操作的圖形介面

## User Journeys

### 文字翻譯

| Context | Action | Outcome |
|---------|--------|---------|
| 使用者開啟應用程式 | 輸入文字、選擇來源語言與目標語言，按下翻譯 | 系統顯示翻譯結果 |

### SRT 字幕翻譯

| Context | Action | Outcome |
|---------|--------|---------|
| 使用者有一份英文 SRT 字幕檔 | 上傳 SRT 檔案、選擇目標語言、選擇性上傳詞彙表，按下翻譯 | 系統產生翻譯後的 SRT 檔案供下載，時間軸不變 |

### 詞彙表管理

| Context | Action | Outcome |
|---------|--------|---------|
| 使用者希望特定詞彙有固定翻譯 | 上傳 CSV 格式詞彙表（來源詞, 目標詞） | 系統在後續翻譯中優先使用詞彙表中的譯詞 |

## Behaviors

### 模型載入

- 系統啟動時載入 `google/translategemma-4b-it` 模型
- 系統透過環境變數 `HF_TOKEN` 驗證 Hugging Face 存取權限
- 模型在首次使用時下載並快取至本地

### 語言選擇

- 系統提供 TranslateGemma 支援的所有語言對供選擇
- 來源語言與目標語言不得相同

### 文字翻譯

- 使用者輸入文字，選擇來源與目標語言，按下翻譯後系統輸出翻譯結果
- 無硬性輸入長度限制，系統盡可能處理大量文字
- 當輸入超過模型上下文長度時，系統自動以 Sliding Window 方式分段翻譯，確保段落間上下文連續性
- 翻譯結果以完整文字呈現，使用者可複製

### SRT 翻譯與上下文

- 系統解析 SRT 檔案，保留原始時間軸與序號
- 翻譯每句字幕時，系統將前後 N 句字幕作為上下文一併送入模型（N 預設為 3，使用者可調整，範圍 0–10）
- 檔案開頭與結尾的字幕，上下文自動縮減至可用範圍
- 翻譯過程中，系統顯示進度指示（已完成句數 / 總句數）
- 翻譯完成後，系統產生新的 SRT 檔案，格式與原檔一致

### 詞彙表

- 詞彙表格式為 CSV，規格如下：

| 項目 | 定義 |
|------|------|
| 編碼 | UTF-8 |
| 分隔符 | 逗號 |
| 標頭列 | 無，第一列即為資料 |
| 欄位 | 第一欄：來源語言詞彙，第二欄：目標語言詞彙 |

- 系統在組成翻譯提示時，將匹配到的詞彙對照附加至提示中，指引模型使用指定譯詞
- 詞彙匹配以完整詞彙比對為主，不做部分字串匹配
- 詞彙表為選填，未提供時系統正常翻譯

### GUI 介面

- 系統提供基於瀏覽器的圖形介面
- 預設使用 Gradio 框架
- 系統在介面上顯示目前使用的運算裝置資訊：
  - 裝置類型（如 NVIDIA RTX 4090、Apple M2 GPU、CPU）
  - 可用 VRAM（GPU）或可用記憶體（CPU）
- 使用 CPU 運算時，系統額外顯示提示：翻譯速度可能較慢

Decided:
- 前端框架：Gradio（適合快速建構 ML 模型的互動式 Web 介面，降低前端開發成本）
- Daggr 不適用於本專案（Daggr 為工作流程編排工具，非 UI 框架；本專案為單一模型翻譯工具，Gradio 已涵蓋所有 UI 需求）

## Error Scenarios

| 情境 | 系統行為 |
|------|---------|
| `HF_TOKEN` 未設定或無效 | 啟動時顯示明確錯誤訊息，引導使用者設定 Token |
| 上傳的檔案非合法 SRT 格式 | 顯示格式錯誤訊息，拒絕處理 |
| 來源語言與目標語言相同 | 禁止送出翻譯請求 |
| 文字翻譯輸入內容為空白 | 禁止送出翻譯請求 |
| 詞彙表 CSV 格式錯誤 | 顯示格式錯誤訊息，提示正確格式 |
| 模型下載失敗（網路問題） | 顯示下載失敗訊息，提示檢查網路連線與 Token |
| 翻譯過程中發生錯誤 | 顯示錯誤訊息，使用者可下載已完成部分的翻譯結果，亦可從中斷處重新嘗試翻譯 |
| GPU 偵測失敗或不支援 | 自動退回 CPU 模式，介面顯示目前使用 CPU 運算 |

## Terminology

| 術語 | 定義 |
|------|------|
| TranslateGemma | Google 開放權重翻譯模型（4B 參數版本） |
| SRT | SubRip Subtitle 格式，純文字字幕檔案格式 |
| HF_TOKEN | Hugging Face 平台的 API 存取權杖 |
| 詞彙表 | CSV 格式的來源詞與目標詞對照表 |
| 上下文窗口 | 翻譯單句字幕時，一併提供給模型的前後鄰近字幕 |
| 運算裝置 | 執行模型推論的硬體，可為 GPU（含 VRAM）或 CPU（使用系統記憶體） |
| Sliding Window | 當文字超過模型上下文長度時，將文字分段並保留段落間重疊部分以維持上下文連續性的翻譯策略 |

## Environment

| 項目 | 需求 |
|------|------|
| Python | >= 3.12 |
| 作業系統 | macOS、Windows |
| 套件管理 | uv（主要）、pip + requirements.txt（備選） |
| 模型 | google/translategemma-4b-it（需 HF_TOKEN） |
| 最低 GPU | NVIDIA GTX 3050（4GB VRAM） |
| 無 GPU 時 | 系統退回 CPU 模式，功能不受限，速度較慢 |

### 硬體適應

- 系統以 4GB VRAM 的低階 GPU（如 GTX 3050）作為基準硬體目標
- 在 VRAM 不足以載入完整模型時，系統自動以降低精度方式載入模型（如量化），使模型可在有限 VRAM 中運行
- 無可用 GPU 時，系統自動退回 CPU 模式，所有功能維持可用
